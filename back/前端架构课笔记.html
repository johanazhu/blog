原生脚手架开发痛点分析

  • 痛点一：重复操作
    • 多 Package 本地link
    • 多 Package 依赖安装
    • 多 Package 单元测试
    • 多 Package 代码提交
    • 多 Package 代码发布
  • 痛点二：版本一致性
    • 发布时版本一致性
    • 发布后相互依赖版本升级

Package 越多。管理复杂度越高

Lerna 简介

Lerna 是一个优化基于 git + npm 的多 package 项目的管理工具

优势
  • 答复减少重复操作
  • 提升操作的标准化

Lerna 是架构优化的产物，它解释了一个架构真理：项目复杂度提升后，就需要对项目进行架构优化。架构优化的主要目标往往都是以效能为核心
